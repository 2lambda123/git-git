but-whatchanged(1)
==================

NAME
----
but-whatchanged - Show logs with difference each cummit introduces


SYNOPSIS
--------
[verse]
'but whatchanged' <option>...

DESCRIPTION
-----------

Shows cummit logs and diff output each cummit introduces.

New users are encouraged to use linkbut:but-log[1] instead.  The
`whatchanged` command is essentially the same as linkbut:but-log[1]
but defaults to show the raw format diff output and to skip merges.

The command is kept primarily for historical reasons; fingers of
many people who learned Git long before `but log` was invented by
reading Linux kernel mailing list are trained to type it.


Examples
--------
`but whatchanged -p v2.6.12.. include/scsi drivers/scsi`::

	Show as patches the cummits since version 'v2.6.12' that changed
	any file in the include/scsi or drivers/scsi subdirectories

`but whatchanged --since="2 weeks ago" -- butk`::

	Show the changes during the last two weeks to the file 'butk'.
	The "--" is necessary to avoid confusion with the *branch* named
	'butk'

GIT
---
Part of the linkbut:but[1] suite
